## Axis audit and honest copy

The engine's real fields are independent `-100..100` axes:

- `rootedAxis` moves toward rooted/community or extractive behavior;
- `relationalAxis` moves toward relational or algorithmic behavior.

They are not a strict inverse pair. Existing choices may move one axis or both: the No-Questions resolution changes `rootedAxis`, the Luxury Tip changes `relationalAxis`, and some relationship scenes also add smaller relational changes. W2-05 therefore does not use the bible's literal “maximizing one lowers the other” slogan and does not change the math.

The permanent phone read model makes presentation-only `0..100` projections over those live fields:

```text
metric_x / Community Trust = round((rootedAxis + 100) / 2)
Platform Efficiency        = round((100 - relationalAxis) / 2)
```

These values are never persisted as new metrics. The on-screen relationship copy is: “Not a strict seesaw: Efficiency reflects what the app rewards; metric_x reflects what the street remembers. A choice may move one, both, or neither.”
